# 크래시 리포팅 조사 — A2 산출물 (운영 준비물)

## 비교 (핵심)
| 방식 | 서버비용 | 데이터 외부전송 | 자동/수동 |
|---|---|---|---|
| Android Vitals | 0원 | Google에 OS 차원 전송(옵트인) | 자동. 단 설치수 적으면 "데이터 부족"으로 무용 |
| Firebase Crashlytics | 0원(무제한 무료) | **Google로 자동 전송** → "온디바이스·전송 0" 원칙 정면 충돌 | 완전 자동 |
| ACRA + acra-mail | 0원 | 사용자 메일 클라이언트 경유(제3자 서버 없음) | 반자동. 라이브러리는 기능개발 정체(2025.9 v5.13.1 마지막) |
| **커스텀 로컬 로그 + 수동 공유** ★ | 0원 | **사용자가 공유 버튼 누르기 전 전송 0** | 완전 수동 |

## 권장
- **UncaughtExceptionHandler → 앱 내부 저장소에 스택트레이스 저장 + 설정에 "버그 리포트 공유"(ACTION_SEND)** — 표준 API만, SDK 0, 원칙 완전 정합.
- Vitals는 Play 배포 시 공짜로 따라오지만 초기 소규모 설치에선 무용(공식 임계값 비공개, 프라이버시 임계).
- Crashlytics 채택 시 Data Safety "No data collected" 선언 불가해짐 — 도입하지 말 것.
