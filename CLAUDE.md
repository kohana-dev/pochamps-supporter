# 포챔스 서포터 앱 (Pokémon Champions Support App)

## 프로젝트 개요
포켓몬 챔피언스(포챔스) 대전 중, 화면에 뜬 상대 포켓몬을 인식해서
그 포켓몬의 **타입 / 특성 / 주요 기술**을 게임 위에 오버레이로 실시간 표시해주는 앱.

## 핵심 결정 사항 (조사 완료)

### 대상 플랫폼: 안드로이드 전용
- 포켓몬 챔피언스 출시: Nintendo Switch(2026-04), iOS·Android(2026-06-17)
- "게임 위 실시간 오버레이"는 **안드로이드에서만 가능**.
  - 안드로이드: `MediaProjection`(화면 캡처) + `SYSTEM_ALERT_WINDOW`("다른 앱 위에 표시") 조합으로 표준 구현 가능.
  - **iOS: 불가능** — 시스템이 앱 간 플로팅 오버레이를 원천 차단. 이 컨셉은 iOS에서 만들 수 없음.
  - Switch: 기기 자체로는 불가능(캡처카드+PC 필요).
  - PC 에뮬레이터(MEmu 등): 오버레이 구현이 가장 쉬움 (참고용).

### 인식 방식: 이미지 분류가 아니라 OCR
- 포챔스는 상대 포켓몬 **이름 텍스트**가 화면에 표시됨.
- 생김새를 CNN으로 분류하는 것보다, 캡처 이미지에서 **이름 영역만 OCR**하는 것이 훨씬 정확하고 간단.
  (메가진화/폼 변화/각도 문제 회피)
- OCR은 **온디바이스** 사용 (Google ML Kit on-device 또는 Tesseract). 클라우드 OCR은 비용 발생하므로 지양.

### 데이터: 로컬 DB
- 이름 → 타입/특성/주요기술 매칭은 **앱에 내장된 로컬 DB**로 처리 (인터넷 불필요).
- 포챔스 포켓몬 풀이 한정적 → 데이터 용량 작음(수 MB), 앱에 동봉 가능.
- 참고 데이터 소스: PokeAPI, 커뮤니티 플랫폼 [포케챔스(yodams)](https://pokemon.yodams.com/).

### 비용 구조
- **상시 서버 비용: 0원** (모든 처리가 온디바이스).
- 데이터 갱신은 앱 업데이트 또는 정적 JSON 호스팅(무료 티어)로 충분.
- Google Play 개발자 등록: 최초 1회 $25.
- (iOS는 대상 아니므로 Apple $99 불필요.)

## 리스크 / 주의
- 경쟁 대전 게임이라 실시간 보조는 이용약관/안티치트 관점에서 회색지대.
- **"화면에 공개된 정보 표시"에만 한정**할 것. 게임 메모리 읽기·자동입력 금지(제재 위험).

## MVP 범위 (초안)
1. `SYSTEM_ALERT_WINDOW` 권한으로 플로팅 오버레이 창 띄우기
2. `MediaProjection`으로 화면 캡처
3. 상대 포켓몬 이름 영역 크롭 → 온디바이스 OCR
4. 로컬 DB 조회 → 타입/특성/주요기술 오버레이에 표시

## 기술 스택 (제안 — 확정 필요)
- 언어/프레임워크: Kotlin (네이티브 안드로이드) 권장. (Flutter도 가능하나 오버레이+캡처는 네이티브 플러그인 필요)
- OCR: Google ML Kit Text Recognition (on-device)
- DB: 앱 내장 JSON 또는 SQLite/Room

## 열린 질문
- 오버레이 UI/UX 형태 (작은 칩 vs 상세 카드)
- 이름 인식 실패 시 fallback (수동 검색?)
- 데이터 최신화 주기 및 방법
